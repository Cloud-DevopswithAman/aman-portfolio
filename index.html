signature project

Agentic Kubernetes Troubleshooting Assistant

An AI agent that investigates live Kubernetes clusters, grounds its reasoning in past incident history through RAG, and exposes its diagnostic tools to any MCP-compatible client — turning a 20-minute manual triage into a single command.

Instead of an engineer manually stitching together kubectl describe, logs, and the events feed, the agent runs that whole investigation itself: it pulls live pod, log, event, deployment, and network signals through a read-only Kubernetes inspection module, retrieves similar past incidents through a Retrieval-Augmented Generation (RAG) pipeline built on Azure AI Search, and reasons over all of it with an Azure OpenAI-backed LLM. The result is a structured diagnosis — root cause, a confidence score, and a concrete remediation step — grounded in real cluster evidence and prior resolution patterns, not just the model's own memory.

architecture — data flow

Kubernetes cluster

Pods, deployments, services, events, and logs — the live evidence layer.

Investigation layer

Read-only Kubernetes inspection — pod status, log collection, event analysis, deployment health, and network/selector checks — run through a FastAPI orchestration layer.

RAG retrieval

Historical incident reports and RCA docs, indexed with Azure OpenAI embeddings in Azure AI Search, are retrieved to ground the diagnosis in real past resolutions.

LLM reasoning layer

Combines live cluster evidence with retrieved incident context to produce a root cause, a confidence score, and step-by-step remediation commands.

MCP server

Exposes live cluster inspection and historical incident lookup as callable tools — any MCP-compatible LLM client can invoke diagnostics directly.

Frontend

Kubeconfig context selection, investigation triggering, live result visualization, and an in-memory history layer for auditability.
